## Description
As an SRE, I want runbooks for common incidents so that I can respond quickly and consistently.

## Components
- Docs: Markdown runbooks stored in the repo.
- Backend: Links from alerts to runbook locations.

## Acceptance Criteria
- Runbooks exist for latency regression triage, feed gap investigation, consumer overload containment, and rollback/canary disable.
- Runbooks include step-by-step procedures and required dashboards.
- Runbooks are reviewed and updated after incidents.
